# credit-card-fraud-detection
Exploratory data analysis of 284,000+ credit card transactions to detect fraud patterns using Python and Pandas


## Overview
This project analyzes a real-world credit card transaction dataset to identify patterns associated with fraudulent activity. Using Python and exploratory data analysis techniques, the project examines transaction behavior, class imbalance, and statistical indicators that distinguish legitimate transactions from fraud — skills directly applicable to risk monitoring and financial data analysis roles.

## Tools & Technologies
- Python (Pandas, NumPy, Matplotlib)
- VS Code
- Data cleaning & transformation
- Exploratory Data Analysis (EDA)
- Statistical analysis & visualization

## Project Goals
- Load and explore a large-scale financial transaction dataset (284,000+ records)
- Analyze class imbalance between fraudulent and legitimate transactions
- Identify statistical patterns and anomalies in transaction amounts and frequency
- Visualize fraud distribution and key risk indicators using charts and graphs
- Summarize findings in a way that supports real-world fraud risk decisions

## Key Findings
- - 73.6% of fraudulent transactions are under $100, suggesting deliberate card testing behavior
- Fraudulent transactions average $122.21 vs $88.29 for legitimate ones — a 38% difference
- Fraud occurs evenly across all time periods with no peak hours, making time-based detection unreliable

## Dataset
- Source: [Credit Card Fraud Detection — Kaggle (MLG-ULB)](https://www.kaggle.com/datasets/mlg-ulb/creditcardfraud)
- 284,807 transactions | 492 fraud cases | 30 features
- Highly imbalanced dataset: fraud represents ~0.17% of all transactions

## How to Run
1. Clone this repository
2. Download the dataset from Kaggle and place `creditcard.csv` in the project folder
3. Open `credit_card_fraud_analysis.py` in VS Code
4. Run the script to generate analysis and visualizations
